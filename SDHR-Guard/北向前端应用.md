# 一、SDN DHR防御系统项目介绍

## 项目概述

这是一个基于 Web 的 SDN 控制器异构冗余防御系统，使用 Vue 2 开发的前端应用。

## 功能需求

- 控制器管理

~~~
显示三种控制器（RYU、POX、OpenDaylight）的状态

支持控制器的启动/停止操作

实时显示控制器健康状态
~~~

- 监控面板

~~~
实时流量监控图表

控制器性能指标

系统健康状态

异常警报显示
~~~

- DHR功能

~~~
异构性：展示三种不同控制器的运行状态
冗余性：显示主备控制器状态
动态性：控制器切换操作和状态展示
~~~

- 防御功能

~~~
异常流量检测结果展示
防御策略配置界面
防御效果实时反馈
~~~

## 技术栈

~~~

## 核心功能的设计

- 流量监控系统

~~~
A. 流量数据采集
   - 端口统计信息
   - 流表统计
   - 数据包分析
   - 带宽利用率

B. 异常检测
   - 流量突变检测
   - DDoS攻击识别
   - 异常连接模式
   - 协议异常分析

C. 监控指标
   - 流量速率
   - 连接数量
   - 包大小分布
   - 协议分布
~~~

- DHR架构

~~~
D (Diversity) - 异构性：使用三种不同的控制器
- RYU (Python实现)
- POX (Python实现)
- OpenDaylight (Java实现)

H (Heterogeneity) - 多样性：
- 不同的实现语言
- 不同的处理逻辑
- 不同的配置参数

R (Redundancy) - 冗余性：
- 同时运行多个控制器
- 主备切换机制
- 状态同步

这个设计实现了：
三种控制器的异构部署
实时监控和切换机制
状态管理和同步
故障恢复流程
~~~

- 裁决机制

~~~
A. 投票裁决
   - 多数表决
   - 加权投票
   - 信任度评估

B. 裁决指标
   - 流表一致性
   - 路由决策
   - 安全策略执行
   - 性能指标

C. 裁决流程
   1. 收集决策
   2. 比较分析
   3. 投票裁决
   4. 执行结果
~~~

- 调度策略

~~~
A. 调度算法
   - 轮询调度
   - 负载均衡
   - 性能感知
   - 安全感知

B. 切换触发条件
   - 性能下降
   - 安全威胁
   - 故障检测
   - 定期轮换

C. 调度流程
   1. 状态检查
   2. 策略选择
   3. 切换准备
   4. 执行切换
   5. 验证结果
~~~

- 测试系统

~~~
A. 白盒插桩
   - 代码插装点

   - 状态监控
   - 错误注入

B. 测试场景
   - 正常流量测试

step2:完善控制器功能-实现的健康检查功能，因为：

- DHR系统的核心需求

~~~
- 我们的系统需要实现控制器的动态切换
- 需要确保控制器真实可用，而不是仅仅进程启动
- 这是实现可靠冗余（Redundancy）的基础
~~~

- 实际应用场景

~~~
- 控制器进程可能启动了，但服务并未就绪
- 控制器可能因各种原因无响应
- 如果没有健康检查，可能导致切换到一个实际不可用的控制器
~~~

- 对后续功能的影响

~~~
- 裁决机制需要依赖控制器的健康状态
- 调度策略需要基于控制器的可用性做决策
- 系统监控需要准确的控制器状态信息
~~~
